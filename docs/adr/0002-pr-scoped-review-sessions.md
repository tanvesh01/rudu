# Keep Review Sessions scoped to pull requests

Rudu keeps one Review Session and Review Chat for a pull request instead of creating a new session for every head SHA. When Rudu detects a newer pull request head, it shows a refresh action in chat and waits for the developer to start a Revision Refresh; that refresh updates the Review Workspace in place, keeps the existing AI runtime when no turn is active, sends the AI a hidden Revision Refresh Notice, and adds an informational Revision Checkpoint for the developer. This preserves conversational continuity while avoiding automatic workspace mutation under an active review and avoiding per-push session proliferation.
